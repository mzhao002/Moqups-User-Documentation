# Glossary

| Term | Definition |
|------|------------|
| Moqups | An online diagramming tool that allows users to create, organize, and collaborate on UML Class Diagrams. |
| UML | A standard modeling language used to visually represent the structure of a system. |
| UML Class Diagram | A type of diagram that shows classes, their attributes, methods, and relationships. |
| Class | A blueprint that defines attributes and methods of an object. |
| Attribute | A variable that stores data in a class. |
| Method | A function that defines the behavior of a class. |
| Relationship | A connection between classes, such as association or inheritance. |
| Connector | A line used to link classes and represent relationships. |
| Workspace | The area where you create and edit diagrams in Moqups. |
| Collaborator | A user who has access to a project. |
| Permission | The level of access given to a collaborator (e.g., edit or comment). |
| Export | The process of saving a diagram as a file (e.g., PNG or PDF). |
| Shareable Link | A link that allows users to access a diagram. |
