# Glossary

| Term | Definition |
|------|-----------|
| Moqups | A web-based diagramming tool used to create UML Class Diagrams. |
| UML | A standard way to visually represent the structure of a system. |
| UML Class Diagram | A diagram that shows classes, their attributes, methods, and relationships. |
| Class | A blueprint that defines attributes and methods of an object. |
| Attribute | A variable that stores data in a class. |
| Method | A function that defines the behavior of a class. |
| Relationship | A connection between classes, such as association or inheritance. |
| Connector | A line used to link classes and represent relationships. |
| Workspace | The area where you create and edit diagrams in Moqups. |
| Page | A canvas inside a project used to organize diagram content. |
| Project | A container that holds your diagrams and pages. |
| Collaborator | A user who has access to your project. |
| Permission | The level of access given to a collaborator (e.g., edit or comment). |
| Export | The process of saving your diagram as a file (e.g., PNG or PDF). |
| Shareable Link | A link that allows others to access your diagram. |
| Starter Plan | A paid plan required to unlock features such as exporting. |
